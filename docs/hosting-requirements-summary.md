# ملخص التطبيق ومتطلبات الاستضافة


---

## 1. نبذة عن التطبيق

تطبيق **Web** لمعاينة المنتجات داخل الغرف:

* تجربة **Room Preview** تربط **شاشة المعرض** بـ**جوال العميل** (يرفع العميل صورة غرفته ويختار المنتج).
* يستخدم **الذكاء الاصطناعي (Gemini)** لإنشاء صورة المعاينة.
* يحتوي على **Seller Chat** للبائعين للاستعلام عن الأصناف والمخزون.
* يحتوي على **لوحة Admin** لإدارة ومراقبة بعض أجزاء النظام.

---

## 2. ممَّ يتكوّن التطبيق؟

| المكوّن | التقنية | الوظيفة |
| ------- | ------- | ------- |
| تطبيق الويب | Next.js / React / TypeScript | الواجهة + الـAPI الخلفي |
| قاعدة البيانات | PostgreSQL | تخزين بيانات التطبيق (بائعون/معارض/جلسات) |
| طبقة الوصول | Prisma | اتصال وإدارة قاعدة البيانات — **ليست قاعدة بيانات بحد ذاتها** |
| Seller Chat backend | **FastAPI** (خدمة منفصلة) | منطق وبيانات المخزون للشات |
| التخزين المؤقت/اللحظي | Redis | حدود الطلبات + التنسيق + الأحداث اللحظية |
| تخزين الملفات | **Object Storage متوافق مع S3** (مثل Cloudflare R2) | حفظ صور الغرف ونتائج المعاينة |
| خدمة الذكاء الاصطناعي | Google **Gemini API** | توليد صورة المعاينة |
| الوصول العام | Domain + HTTPS | الوصول للتطبيق وتوليد رموز QR |

---

## 3. كيف يعمل النظام باختصار؟

```text
المستخدم أو شاشة المعرض
        ↓
تطبيق Next.js
        ↓
PostgreSQL / Redis / Object Storage
        ↓
Gemini للمعاينة
FastAPI لشات البائعين
```

* **الصور والنتائج لا يجب تخزينها على القرص المحلي للسيرفر** — تُحفظ في Object Storage (وإلا تُفقد عند كل إعادة تشغيل/نشر).
* **خدمة FastAPI يجب أن تكون متاحة لتطبيق Next.js** (يتصل بها الخادم لخادم).

---

## 4. ما الذي يجب أن توفّره الشركة؟

### الاستضافة الأساسية
- [ ] بيئة تشغيل تدعم تطبيق Next.js وNode.js.
- [ ] Domain أو Subdomain.
- [ ] HTTPS / SSL.
- [ ] قاعدة بيانات PostgreSQL.
- [ ] Redis.
- [ ] Object Storage متوافق مع S3.
- [ ] استضافة أو عنوان متاح لخدمة FastAPI.
- [ ] اتصال Internet outbound للوصول إلى Gemini.
- [ ] مكان آمن لحفظ Environment Variables وSecrets.
- [ ] نسخ احتياطي أساسي لقاعدة البيانات.
- [ ] Monitoring أو Logs أساسية.

### الشبكة
- [ ] وصول خادم Next.js إلى PostgreSQL.
- [ ] وصوله إلى Redis.
- [ ] وصوله إلى Object Storage.
- [ ] وصوله إلى FastAPI.
- [ ] وصوله إلى Gemini عبر الإنترنت.
- [ ] دعم HTTPS والطلبات طويلة المدة (المعاينة قد تستغرق وقتاً).

## 5. المتطلبات المبدئية للسيرفر

### Staging / تجربة
| المورد | تقدير مبدئي |
| ------ | ----------- |
| CPU | 2 vCPU |
| RAM | 4 GB |
| Disk | 20 GB |
| Instances | 1 |

### Production مبدئي
| المورد | تقدير مبدئي |
| ------ | ----------- |
| CPU | 2–4 vCPU |
| RAM | 4–8 GB |
| Disk | 20–40 GB |
| Instances | 1 كبداية أو 2 للتوافر العالي |

```text
هذه أرقام مبدئية وليست نهائية، ويجب مراجعتها بعد معرفة عدد المستخدمين المتزامنين وإجراء Load Testing.
```
