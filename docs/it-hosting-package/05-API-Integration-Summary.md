# ملخص التكاملات والـAPIs

> ملخص تقني مختصر للتكاملات الموجودة فعلياً في الكود. لا يحتوي على مفاتيح أو أسرار أو قيم فعلية.

## 1. Google Gemini API
| البند | التفصيل |
| ----- | ------- |
| الغرض | توليد صورة معاينة الغرفة بالذكاء الاصطناعي |
| اتجاه الاتصال | خادم Next.js → Gemini (صادر) |
| البروتوكول | HTTPS / REST |
| المصادقة | مفتاح API (server-only) |
| متغيّرات البيئة | `GEMINI_API_KEY`, `GEMINI_IMAGE_MODELS`, `GEMINI_MAX_CONCURRENT`, `GEMINI_CALL_TIMEOUT_MS` |
| المُرسَل | صورة الغرفة + بيانات المنتج/التوجيه |
| المستلَم | صورة المعاينة المولّدة |
| Timeout / Retry | مهلة محدّدة + تحكّم بالتزامن (semaphore) + محاولات إعادة |
| عند الفشل | رسالة فشل آمنة للمستخدم دون كشف تفاصيل |
| الحالة | حالي |

## 2. Object Storage (Cloudflare R2 / S3-compatible)
| البند | التفصيل |
| ----- | ------- |
| الغرض | تخزين صور الغرف ونتائج المعاينة |
| اتجاه الاتصال | خادم Next.js → التخزين؛ والمتصفح → التخزين (رفع مباشر) |
| البروتوكول | HTTPS / S3 API |
| المصادقة | مفاتيح وصول (server-only) + Presigned URL للرفع |
| متغيّرات البيئة | `STORAGE_PROVIDER`, `R2_ENDPOINT`, `R2_BUCKET_NAME`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_PUBLIC_URL` |
| المُرسَل | ملفات الصور |
| المستلَم | روابط الملفات المخزّنة |
| ملاحظات | CORS مطلوب على الـbucket؛ ممنوع التخزين المحلي في الإنتاج |
| عند الفشل | فشل الرفع/القراءة يُعالَج برسالة آمنة |
| الحالة | حالي |

## 3. Redis
| البند | التفصيل |
| ----- | ------- |
| الغرض | حدود الطلبات + أقفال التوليد + الأحداث اللحظية (SSE pub/sub) |
| اتجاه الاتصال | خادم Next.js → Redis |
| البروتوكول | اتصال Redis (TLS مفضّل) |
| المصادقة | عبر سلسلة الاتصال |
| متغيّرات البيئة | `REDIS_URL`, `ENABLE_REDIS` |
| عند الفشل | الميزات تعمل على عملية واحدة فقط بدونه (مطلوب في الإنتاج) |
| الحالة | حالي |

## 4. PostgreSQL
| البند | التفصيل |
| ----- | ------- |
| الغرض | تخزين بيانات التطبيق (بائعون/معارض/جلسات) — لا مخزون |
| اتجاه الاتصال | خادم Next.js → PostgreSQL (عبر Prisma) |
| البروتوكول | اتصال Postgres (TLS) |
| المصادقة | عبر `DATABASE_URL` |
| متغيّرات البيئة | `DATABASE_URL`, `DATABASE_POOL_SIZE` |
| عند الفشل | فشل الطلبات المرتبطة بقاعدة البيانات |
| الحالة | حالي |

## 5. Next.js ↔ FastAPI (Seller Chat backend)
| البند | التفصيل |
| ----- | ------- |
| الغرض | استعلام المخزون + اقتراح أكواد المنتجات |
| اتجاه الاتصال | خادم Next.js → FastAPI (خادم‑لخادم) |
| البروتوكول | HTTPS / REST |
| المصادقة | JWT خارجي قصير العمر (~60 ثانية) |
| متغيّرات البيئة | `CHATBOT_FASTAPI_URL`, `EXTERNAL_SELLER_JWT_SECRET` |
| المُرسَل | نص السؤال + نبرة الرد |
| المستلَم | نص الإجابة + بطاقات المخزون |
| Timeout | مهلة محدّدة للشات والاقتراحات |
| عند الفشل | تصنيف آمن (الخدمة غير متاحة / مهلة / خطأ upstream) دون كشف التفاصيل |
| الحالة | حالي |

## 6. Admin ↔ FastAPI (الإدارة)
| البند | التفصيل |
| ----- | ------- |
| الغرض | استيراد المخزون + حالة ومقاييس الشات |
| اتجاه الاتصال | خادم Next.js (Admin) → FastAPI (خادم‑لخادم) |
| البروتوكول | HTTPS / REST |
| المصادقة | JWT داخلي للإدارة (منفصل عن رمز البائع) |
| متغيّرات البيئة | `CHATBOT_FASTAPI_URL`, `INTERNAL_JWT_SECRET` |
| عند الفشل | رسالة حالة آمنة في لوحة الإدارة |
| الحالة | حالي |

## 7. Health Checks (المراقبة الداخلية)
| البند | التفصيل |
| ----- | ------- |
| الغرض | فحص صحة التطبيق والخدمات (DB/Redis/Storage/Gemini/FastAPI) |
| اتجاه الاتصال | داخلي + مراقبة خارجية (Uptime) |
| المسارات | `/api/health`, `/api/admin/system-health` |
| المصادقة | الصحة العامة مفتوحة؛ صحة النظام محمية بالإدارة |
| الحالة | حالي |

## 8. GitHub + بيئة النشر
| البند | التفصيل |
| ----- | ------- |
| الغرض | إدارة الكود والنشر |
| البناء/التشغيل | `prisma generate && next build` ثم `next start` |
| متغيّرات البيئة | `SENTRY_ORG`, `SENTRY_PROJECT`, `NEXT_PUBLIC_SENTRY_DSN` |
| ملاحظة | تفاصيل المنصّة والـCI/CD تحتاج تأكيد من فريق IT |
| الحالة | حالي |

## 9. SAP (تخطيط مستقبلي فقط)
| البند | التفصيل |
| ----- | ------- |
| الغرض | مصدر مخزون معتمد مستقبلاً |
| الحالة | **غير مطبّق** — لا اتصال أو إعداد SAP في الكود |
| ملاحظة | عند البدء يلزم تحديد طريقة الاتصال ومسار شبكة آمن (يحتاج فريق IT وSAP) |

> لا تُعرض في هذه الوثيقة أي مفاتيح API أو أسرار JWT أو كلمات مرور أو قيم فعلية أو بيانات عملاء.
